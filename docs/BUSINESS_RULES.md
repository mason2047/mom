# MOM孕期助手 - 业务规则文档

## 1. 孕周计算规则

### 1.1 基于末次月经(LMP)计算
```
当前孕周 = (今天日期 - LMP日期) / 7
孕天数 = (今天日期 - LMP日期) % 7
格式显示: "孕X周X天"
```

### 1.2 预产期(EDD)计算 - Naegele公式
```
EDD = LMP + 280天
或者: 月份 - 3（或 + 9），日期 + 7
示例: LMP = 2025-11-25 → EDD = 2026-09-01
```

### 1.3 距预产期天数
```
距预产期 = EDD - 今天日期（天数）
```

### 1.4 孕期三阶段划分
| 阶段 | 英文 | 孕周范围 | 显示名称 |
|------|------|----------|----------|
| 孕早期 | first | 1~13+6周 | 孕早期 |
| 孕中期 | second | 14~27+6周 | 孕中期 |
| 孕晚期 | third | 28~40周 | 孕晚期 |

### 1.5 孕程进度百分比
```
进度% = 当前总天数 / 280 * 100
上限 100%
```

## 2. BMI 计算与分类

### 2.1 BMI 公式
```
BMI = 体重(kg) / (身高(m))^2
示例: 55kg / (1.65m)^2 = 20.2
结果保留一位小数
```

### 2.2 BMI 分类（WHO标准）
| BMI 范围 | 分类 | 中文名 |
|----------|------|--------|
| < 18.5 | underweight | 偏瘦 |
| 18.5 ~ 24.9 | normal | 正常 |
| 25.0 ~ 29.9 | overweight | 偏胖 |
| >= 30.0 | obese | 肥胖 |

## 3. 孕期体重增长推荐（IOM 2009标准）

### 3.1 全孕期推荐总增重

| BMI 分类 | 单胎(kg) | 双胎(kg) |
|----------|----------|----------|
| 偏瘦 | 12.5 ~ 18.0 | — |
| 正常 | 11.5 ~ 16.0 | 16.8 ~ 24.5 |
| 偏胖 | 7.0 ~ 11.5 | 14.1 ~ 22.7 |
| 肥胖 | 5.0 ~ 9.0 | 11.4 ~ 19.1 |

**参考标准来源:** IOM (Institute of Medicine) 2009 Guidelines

### 3.2 每周推荐增重（孕中晚期）

| BMI 分类 | 每周推荐(kg/周) |
|----------|----------------|
| 偏瘦 | 0.44 ~ 0.58 |
| 正常 | 0.35 ~ 0.50 |
| 偏胖 | 0.23 ~ 0.33 |
| 肥胖 | 0.17 ~ 0.27 |

**说明:**
- 孕早期（1~13周）总增重 0.5~2.0 kg，不按周计算
- 孕中晚期（14~40周）按每周推荐增重

### 3.3 当前增重评估逻辑
```
推荐范围 = 按当前孕周线性插值计算

如果 孕周 <= 13:
    推荐区间 = [0.5, 2.0]
否则:
    基础 = 上面的孕早期增重中位值(1.0)
    额外周数 = 当前孕周 - 13
    推荐下限 = 基础 + 额外周数 * 每周推荐下限
    推荐上限 = 基础 + 额外周数 * 每周推荐上限

评估结果:
- normal: 当前增重在推荐区间内
- high: 当前增重超过推荐上限
- low: 当前增重低于推荐下限
```

## 4. 血压评估规则（孕期标准）

### 4.1 血压分级
| 等级 | 收缩压(mmHg) | 舒张压(mmHg) | 评估 | 处理建议 |
|------|-------------|-------------|------|----------|
| 正常 | < 120 | 且 < 80 | normal | 继续监测 |
| 偏高 | 120~139 | 或 80~89 | elevated | 注意休息，复测 |
| 高血压 | >= 140 | 或 >= 90 | high | 尽快就医 |
| 严重高血压 | >= 160 | 或 >= 110 | critical | 立即就医 |

### 4.2 评估逻辑
```
取收缩压和舒张压中较严重的等级
例: 128/82 → 收缩压偏高 + 舒张压偏高 → 评估: elevated
```

### 4.3 异常提醒触发条件
- 单次 >= 140/90: 立即推送提醒
- 连续2次 >= 120/80: 推送关注提醒
- 连续3次偏高: 强烈建议就医提醒

## 5. 血糖评估规则（GDM标准）

### 5.1 孕期血糖阈值（ADA / IADPSG标准）

| 测量时机 | 正常值 | 异常值 | 单位 |
|----------|--------|--------|------|
| 空腹 | < 5.1 | >= 5.1 | mmol/L |
| 餐后1小时 | < 10.0 | >= 10.0 | mmol/L |
| 餐后2小时 | < 8.5 | >= 8.5 | mmol/L |

### 5.2 GDM 诊断标准（OGTT 75g糖耐量试验）
- 空腹 >= 5.1 mmol/L
- 1小时 >= 10.0 mmol/L
- 2小时 >= 8.5 mmol/L
- **以上任意一项达到即可诊断 GDM**

### 5.3 评估逻辑
```
根据 meal_type 取对应阈值
- fasting: threshold = 5.1
- post_1h: threshold = 10.0
- post_2h: threshold = 8.5

if value < threshold:
    evaluation = "normal"
elif value < threshold * 1.2:  // 超标 20% 以内
    evaluation = "elevated"
else:
    evaluation = "high"
```

## 6. 胎动计数规则

### 6.1 适用孕周
- 建议从 **孕28周** 开始记录
- 28周前显示"暂不可用"

### 6.2 正常标准
| 时间段 | 正常次数 | 异常 |
|--------|----------|------|
| 1小时 | >= 3~5次 | < 3次 |
| 2小时 | >= 10次 | < 10次 |
| 12小时 | >= 30次 | < 20次 |

### 6.3 评估逻辑
```
if duration_minutes >= 60:
    if total_kicks >= 3:
        evaluation = "normal"
    elif total_kicks >= 1:
        evaluation = "low"    // 偏少，建议继续观察
    else:
        evaluation = "critical"  // 立即就医
```

### 6.4 异常提醒
- 1小时 < 3次: 推送"胎动偏少"提醒
- 连续2次计数 < 3次/小时: 推送"紧急"提醒，建议立即就医

## 14. 胎儿估重规则

### 14.1 Hadlock 公式（估算胎儿体重）

使用 Hadlock IV 公式（基于 AC 和 FL）:
```
log10(EFW) = 1.304 + 0.05281 * AC + 0.1938 * FL - 0.004 * AC * FL
```

**注意:**
- AC（腹围）和 FL（股骨长）需转换为 **cm** 后代入公式
- EFW 结果单位为 **g**（克）
- 此公式误差范围约 +/- 15%

**示例:**
```
AC = 110.3mm = 11.03cm, FL = 25.8mm = 2.58cm
log10(EFW) = 1.304 + 0.05281*11.03 + 0.1938*2.58 - 0.004*11.03*2.58
           = 1.304 + 0.5825 + 0.5000 - 0.1138
           = 2.2727
EFW = 10^2.2727 = 187g (约)
```

### 14.2 胎儿生长百分位评估

基于 WHO 胎儿生长标准（Intergrowth-21st 或 WHO 参考数据），按孕周查找 EFW 对应的百分位:

| 评估 | 百分位范围 | 含义 | 处理建议 |
|------|-----------|------|----------|
| small | < P10 | 偏小 (SGA) | 建议复查，排除胎儿生长受限 |
| normal | P10 ~ P90 | 正常 (AGA) | 继续定期监测 |
| large | > P90 | 偏大 (LGA) | 关注血糖控制，注意巨大儿风险 |

### 14.3 评估逻辑
```
1. 将 B超数据(bpd, hc, ac, fl) 代入 Hadlock 公式 → EFW
2. 根据 gestational_week 查询 WHO 参考数据 → 获取 P10, P50, P90 值
3. 计算 EFW 所在百分位:
   if efw < p10: evaluation = "small"
   elif efw > p90: evaluation = "large"
   else: evaluation = "normal"
4. 精确百分位使用线性插值或 Z-score 计算
```

### 14.4 B超报告 OCR 自动提取

当 source = "ocr" 时:
```
1. OCR 识别 B超报告图片
2. 正则匹配提取关键数值:
   - BPD: /BPD[:\s]*(\d+\.?\d*)\s*mm/i
   - HC:  /HC[:\s]*(\d+\.?\d*)\s*mm/i
   - AC:  /AC[:\s]*(\d+\.?\d*)\s*mm/i

3. 校验数值范围合理性
4. 提取结果供用户确认后提交
```

## 15. 饮食问卷→模式映射规则

### 15.1 饮食模式自动推导

根据问卷结果自动确定饮食模式:

```
if has_gdm == true:
    if bmi_category in ["overweight", "obese"]:
        mode = "gdm_strict"
    else:
        mode = "gdm_moderate"
elif bmi_category == "obese":
    mode = "gdm_moderate"  // 肥胖孕妇也建议控糖
else:
    mode = "balanced"
```

### 15.2 推荐热量计算

```
基础代谢率 (BMR) = 655 + (9.6 * 孕前体重kg) + (1.8 * 身高cm) - (4.7 * 年龄)

活动系数:
  sedentary: 1.2
  light: 1.375
  moderate: 1.55
  active: 1.725

每日能量需求 (TDEE) = BMR * 活动系数

孕期额外需求:
  孕早期 (1-13周): +0 kcal/天
  孕中期 (14-27周): +300 kcal/天
  孕晚期 (28-40周): +450 kcal/天

推荐热量 = TDEE + 孕期额外需求
最低不低于 1500 kcal/天, 最高不超过 2500 kcal/天
```

### 15.3 营养素配比推导

| 模式 | 碳水 | 脂肪 | 蛋白质 |
|------|------|------|--------|
| balanced | 50-65% | 20-30% | 15-20% |
| gdm_moderate | 40-50% | 25-35% | 20-25% |
| gdm_strict | 30-40% | 30-35% | 25-30% |

### 15.4 饮食限制影响

```
if "lactose_intolerant" in restrictions:
    提示: "推荐豆浆、杏仁奶等替代乳制品"
    自动排除含乳糖食物

if "vegetarian" in restrictions:
    蛋白质推荐来源调整为豆制品、蛋奶类
    额外推荐: 铁剂补充

if food_allergies 不为空:
    在食物推荐和替代中自动排除过敏食物
```

## 16. 替代食物推荐规则

### 16.1 筛选条件
```
1. 相同类别（category 一致）
2. GI 值低于原食物 30% 以上:
   alternative.gi_value <= original.gi_value * 0.7
3. 不在用户过敏列表中
4. 不在用户饮食限制排除列表中
5. pregnancy_safe = true
```

### 16.2 排序规则
```
优先级:
1. GI 降低幅度越大越靠前
2. 热量差异越小越靠前（保证替代后热量摄入基本不变）
3. 有 food_alternatives 表中预定义关系的优先展示
```

### 16.3 动态生成 vs 预定义
- **预定义**: food_alternatives 表存储人工审核的推荐关系，优先使用
- **动态生成**: 如果预定义数据不足，则按筛选条件从 food_database 中实时查询

## 19. 积分每日上限规则

### 19.1 每日获取上限
```
每日积分获取上限: 200 幸孕币/天

不计入上限的行为:
- 注册奖励 (+100)
- 建档奖励 (+100)
- 连续打卡奖励 (+50/+200)
- 成就解锁奖励
- 邀请好友注册/活跃奖励
- 邀请里程碑奖励
```

### 19.2 限额校验逻辑
```
1. 用户完成可获币操作时
2. 查询 Redis: daily_coins:{user_id}:{today_date}
3. if 已获取量 + 本次奖励 > 200:
     实际发放 = max(0, 200 - 已获取量)
     如果实际发放 = 0: 提示"今日积分已达上限"
4. 更新 Redis 计数
5. 写入 coin_transactions 记录
```

## 20. 兑换规则

### 20.1 兑换前校验
```
1. 商品状态检查: status = 1 (available)
2. 库存检查: stock > 0
3. 限购检查: 用户已兑换次数 < limit_per_user
4. 余额检查: 用户积分余额 >= price_coins
5. 时间检查: NOW() between start_at and end_at (如果设置了时间窗口)
```

### 20.2 兑换流程
```
1. 获取分布式锁: shop_stock_lock:{item_id} (防止超卖)
2. 二次校验库存和余额
3. 事务操作:
   a. 扣减用户积分 (coin_transactions: amount = -price_coins)
   b. 减少商品库存 (shop_items: stock -= 1, sold_count += 1)
   c. 创建兑换订单 (redemption_orders)
4. 释放锁
5. 异步:
   a. 实物商品 → 创建发货任务
   b. 虚拟商品 → 直接发放（如优惠券写入用户账户）
```

### 20.3 取消/退款规则
```
- 虚拟商品: 不支持退款
- 实物商品:
  - 未发货: 可取消，全额退还积分
  - 已发货: 不支持取消
  - 已签收: 不支持退款
```
