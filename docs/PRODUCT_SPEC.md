# MOM孕期助手 - 产品规格说明书

> 版本: V1.0 | 更新日期: 2026-03-06 | 文档编号: MOM-PRD-001

---

## 一、产品概述

### 1.1 基本信息

| 项目 | 内容 |
|------|------|
| **产品名称** | MOM孕期助手 |
| **Slogan** | 为你的宝宝保驾护航 |
| **产品形态** | 微信小程序（主）/ H5（辅） |
| **产品定位** | 面向孕妈妈的一站式智能孕期健康管理平台 |

### 1.2 目标用户画像

| 维度 | 描述 |
|------|------|
| **核心用户** | 18-40岁孕期女性，以初产妇为主 |
| **教育水平** | 大专及以上，具备基本健康素养 |
| **设备习惯** | 微信重度用户，习惯小程序交互 |
| **核心痛点** | 产检流程不清晰、报告看不懂、饮食不知如何管理、缺乏专业孕期指导 |
| **次要用户** | 经产妇（多胎经验）、妊娠期糖尿病（GDM）孕妇、高龄孕妇 |

### 1.3 核心价值主张

1. **智能产检管理** -- 标准11次产检自动排期、提醒、记录，不漏一次检查
2. **AI 报告解读** -- 拍照上传检验报告，AI 结合孕周给出通俗易懂的解读
3. **全方位健康监测** -- 体重、血压、血糖、胎动、胎儿估重五维追踪，趋势可视化
4. **饮食智能助手** -- 拍照识别食物、GDM 控糖模式、GI 标注、替代食物推荐
5. **MOM AI 伴侣** -- 24小时智能对话，知识问答、情感支持、个性化建议
6. **月子一站式规划** -- 月子中心/月嫂浏览、对比、预约，产后无缝衔接

### 1.4 产品愿景

成为中国孕妈妈最信赖的数字孕期伴侣，覆盖从建档到产后的完整生命周期，通过 AI 技术让专业的孕期管理触手可及。

## 四、功能模块详细说明

---

### 模块1: 引导与建档

**功能概述:** 用户首次使用时的引导流程，包括微信授权登录和孕期基础信息录入，建档后系统自动初始化所有孕期数据。

**页面列表:**

| 路由 | 页面名称 | 说明 |
|------|---------|------|
| `/` | Splash 首屏 | 品牌展示+登录入口 |
| `/onboarding` | 建档引导页 | 3步表单引导 |

**功能点:**

**F1.1.1 Splash 首屏展示**
展示品牌Logo、Slogan"为你的宝宝保驾护航"、孕妇插画，以及四个核心功能亮点（智能产检、AI报告、宝宝发育、MOM助手），底部为"微信一键登录"主按钮。纯静态展示，无 API 调用。

**F1.1.2 微信授权登录**
点击登录按钮后，底部弹出微信授权弹窗（Bottom Sheet），展示微信Logo、授权说明、"微信授权"绿色按钮和"暂不授权"跳过按钮。授权后调用 `POST /auth/wechat-login`，根据返回的 `has_profile` 字段判断跳转建档页或首页。底部有隐私政策和用户协议链接。

**F1.2.1 建档引导 Step1 -- 基础孕期信息**
录入末次月经日期（LMP），使用滚轮式日期选择器，选择后前端通过 Naegele 公式自动计算预产期（EDD = LMP + 280天），并在 hero 区域实时展示对应孕周的水果对照动画。

**F1.2.2 建档引导 Step2 -- 体格信息**
输入孕前体重（kg）和身高（cm），前端实时计算 BMI 值和分类（偏瘦/正常/偏胖/肥胖），根据 IOM 2009 标准在计算结果区显示推荐增重范围（如"正常 BMI，推荐增重 11.5~16.0 kg"）。

**F1.2.3 建档引导 Step3 -- 孕产情况**
通过 Chip 选择器选择: 初产妇/经产妇、单胎/双胎、高危因素（妊高症史/妊娠糖尿病/高龄/无）。高危因素支持多选，选择"无"时自动清除其他选项。可选填就诊医院。

**F1.2.4 建档成功**
提交后弹出成功弹窗，展示孕周水果对照图、"建档成功"标题、当前孕周信息，以及"开始孕期之旅"按钮跳转首页。后端自动创建11次标准产检计划、产检提醒、默认饮食目标，发放100幸孕币并解锁"初来乍到"成就。

**数据依赖:**
- 输入: LMP日期、孕前体重、身高、孕产类型、胎数、高危因素、医院
- 输出: EDD、BMI、BMI分类、推荐增重范围

**业务规则:**
- BMI = 体重(kg) / (身高(m))^2，保留一位小数
- EDD = LMP + 280天
- BMI分类: <18.5 偏瘦 / 18.5-24.9 正常 / 25-29.9 偏胖 / >=30 肥胖
- 推荐增重: 偏瘦 12.5-18.0kg / 正常 11.5-16.0kg / 偏胖 7.0-11.5kg / 肥胖 5.0-9.0kg（单胎）

### 模块2: 首页 Home

**功能概述:** 作为 Tab 1，是用户每日打开的核心页面，聚合展示当前孕周状态、宝宝发育信息、妈妈健康提示、产检提醒和健康快捷记录入口。

**页面列表:**

| 路由 | 页面名称 |
|------|---------|
| `/home` | 首页 Tab |
| `/home/knowledge` | 孕期知识详情页 |

**功能点:**

**F2.1.1 孕周 Banner**
顶部渐变 Banner 展示核心孕周信息: "孕X周X天"（大字号）、所在孕期阶段（如"孕中期第1周"）、距预产期天数、以及一个小按钮"查看本周变化"链接到知识页。右侧有孕妇轮廓装饰图。背景色为暖橙渐变。

**F2.1.2 气泡动画区**
Banner 下方的交互气泡区: 主气泡（最大）展示胎儿emoji和心跳脉冲动画，有呼吸缩放和脉冲发光效果；中气泡展示当前孕周的水果对照emoji和名称；小气泡展示胎儿发育关键数据（身长/体重）。三个气泡各自有浮动动画。

**F2.2.1 宝宝说卡片**
白色圆角卡片，左侧有水果emoji图标，右侧以第一人称"宝宝口吻"展示 AI 生成的胎儿发育语录，内容基于当前孕周的发育数据（如"妈妈，我现在有8.7cm啦，大概一颗山竹那么大"）。左边框有橙色强调线。

**F2.3.1 妈妈健康卡片**
展示本周妈妈身体变化概要（2-3句），底部有"点击查看本周详情"的引导标签，点击跳转 `/home/knowledge` 知识详情页。

**F2.4.1 产检提醒卡片**
展示下次产检信息: 产检名称、孕周范围、预计日期、检查项目标签。有"查看详情"和"记录产检"两个操作按钮。如无待做产检则显示进度概览。

**F2.5.1 健康快捷记录区**
四宫格快捷入口: 体重、血压、血糖、胎动，每个入口展示最近一次记录值和评估状态（正常/偏高/偏低），点击跳转对应健康记录子页面。异常值卡片有红色边框高亮。

**F2.6.1 妈妈变化弹窗**
用户每日首次进入首页时自动弹出（sessionStorage 控制同会话只展示一次），展示本周妈妈身体可能出现的变化和注意事项，有"我知道了"关闭按钮。

**F2.7.1 孕期知识详情页**
分 Tab 展示: "宝宝发育"和"妈妈健康"。宝宝发育包含发育亮点、身体各部位变化描述；妈妈健康包含身体变化列表、营养提醒、需警惕信号、心理变化提示、本周待办清单（含跳转链接）。用户停留超过30秒计为"阅读完成"，触发每日阅读任务奖励。

**数据依赖:**
- 孕期档案（LMP、EDD、BMI）
- 孕周知识数据（按 week 查询）
- 最近健康记录（体重/血压/血糖各1条）
- 待做产检计划

**业务规则:**
- 孕周 = (今天 - LMP) / 7，孕天 = (今天 - LMP) % 7
- 孕期进度 = 当前天数 / 280 * 100%
- 水果对照表: 按孕周4-40周匹配对应水果（共37种）
- 孕期三阶段: 1-13+6周孕早期 / 14-27+6周孕中期 / 28-40周孕晚期

### 模块3: MOM AI 助手

**功能概述:** 作为 Tab 2，提供 AI 驱动的智能对话能力，涵盖孕期知识问答、产检建议、情感支持和产检报告 AI 解读。

**页面列表:**

| 路由 | 页面名称 |
|------|---------|
| `/assistant` | MOM助手 Tab |
| `/assistant/report-upload` | 报告上传页 |
| `/assistant/report-result/[id]` | 报告解读结果页 |

**功能点:**

**F3.1.1 欢迎区**
页面顶部展示 MOM 品牌形象: 渐变橙色头像、品牌名"MOM AI"、副标题"你的孕期智能伙伴"，以及个性化问候语（如"你好宝妈~现在孕14周3天了"），附带 MOM 助手的能力介绍文本。

**F3.1.2 孕周信息卡**
紧凑信息卡展示当前孕周（"孕14周+3天"）和距预产期天数（"距预产期182天"），左侧有胎儿emoji。为对话提供上下文参考。

**F3.2.1 快捷功能入口**
三宫格快捷按钮: "产检记录"（跳转 `/checkup/record`）、"AI报告解读"（跳转 `/assistant/report-upload`）、"饮食助手"（跳转 `/diet`）。每个按钮有图标和简短描述，背景色区分。

**F3.3.1 AI 智能对话**
核心对话区域，支持文字输入。AI 回复以 SSE 流式方式逐字输出，支持 Markdown 渲染（加粗、列表、换行），气泡样式区分用户（橙色渐变右侧）和 AI（白色左侧），AI 消息带有 MOM 头像。消息末尾可附带操作链接（如"查看产检日历"），以 Chip 样式展示，点击跳转对应页面。每日首次提问奖励+5幸孕币。

**F3.3.2 报告解读入口卡片**
当对话涉及报告解读时，AI 回复中嵌入结构化卡片: 标题"产检报告解读"、描述文字、"马上解读"按钮（跳转报告上传页）。卡片有独立圆角样式和阴影。

**F3.4.1 情感支持对话**
当用户表达焦虑、疲惫等负面情绪时，AI 切换为温暖关怀模式: 先共情回应（"抱抱你，这种感受非常正常"），再给出孕期心理知识（"孕14周前后很多妈妈都会经历情绪缓冲期"），最后引导用户倾诉具体困扰。不直接给医学建议，遇紧急情况优先建议就医。

**F3.5.1 报告上传页**
选择报告类型（血常规/尿常规/肝肾功/糖耐量/甲功/超声/唐筛/其他），支持从相册选择或拍照上传报告图片（最多5张，单张最大10MB）。上传完成后点击"开始AI解读"按钮。展示上传进度和缩略图预览。

**F3.5.2 报告解读结果页**

**数据依赖:**

### 模块5: 健康记录

**功能概述:** 提供体重、血压、血糖、胎动、胎儿估重五维健康数据的记录、趋势分析和异常预警，帮助孕妈妈全面追踪孕期健康状态。

**功能点:**

**F5.1.1 综合看板**
四宫格数据概览卡片: 体重（最新值+较上次变化）、血压（最新收缩压/舒张压+评估状态）、血糖（空腹最新值）、胎儿估重（最新估重g）。异常卡片有红色边框高亮。下方为体重趋势折线图（可切换时间范围: 2周/1月/孕期），以及四个快速记录入口按钮。

**F5.2.1 体重记录**
底部弹出输入Sheet: 大号数字输入框（48px字号橙色）配"kg"单位标注，自动计算与上次差值和孕期总增重。保存后评估是否在 IOM 推荐范围内，返回 normal/high/low 状态和文案提示。

**F5.2.2 体重趋势图**
折线图展示历史体重记录，叠加显示 IOM 推荐增重曲线（上限/下限区间，用浅色填充）。图例标注"实际体重"和"推荐范围"。支持切换时间范围（2周/1月/孕早期/孕中期/孕晚期/全孕期）。下方为记录列表，每条记录展示日期、体重值、变化量、评估 Badge。

**F5.3.1 血压记录**
双输入框并排（收缩压/舒张压），中间有"/"分隔符。支持选择测量体位（坐姿/卧姿/站姿）。保存后按孕期标准四级评估: 正常(<120/80) / 偏高(120-139/80-89) / 高血压(>=140/>=90) / 严重高血压(>=160/>=110)。

**F5.3.2 血压异常提醒**
单次>=140/90立即推送提醒；连续2次>=120/80推送关注提醒；连续3次偏高推送强烈建议就医提醒。评估结果在记录保存后以 Badge 和提示文案展示。

**F5.4.1 血糖记录**
单输入框配"mmol/L"单位，需先选择测量时机（空腹/餐后1小时/餐后2小时），使用对应 GDM 阈值评估: 空腹<5.1 / 餐后1h<10.0 / 餐后2h<8.5。异常值显示红色提示。

**F5.5.1 胎动计数**
计时器界面: 大圆形"点击记录"按钮（每次点击记录一次胎动），上方显示已计时时长和已记录次数。支持手动开始和结束，结束后评估: 1小时>=3次正常 / <3次偏少（建议继续观察）/ 0次危急（建议就医）。仅建议孕28周后使用，28周前显示"暂不可用"提示。历史记录列表展示每次计数的时长、次数、评估。

**F5.6.1 胎儿估重**
四个输入框录入B超数据: BPD（双顶径mm）、HC（头围mm）、AC（腹围mm）、FL（股骨长mm）。支持手动输入或 OCR 拍照自动提取。提交后使用 Hadlock IV 公式计算估重（EFW），根据 WHO 标准查询百分位（P10/P50/P90），评估 small(<P10)/normal(P10-P90)/large(>P90)。

**F5.6.2 胎儿生长曲线**
折线图展示历史估重记录，叠加 WHO 参考曲线（P10/P50/P90 三条线），用户数据点标注在曲线上。直观展示胎儿发育是否在正常区间。

**数据依赖:**
- 孕期档案（孕前体重、BMI、当前孕周）
- 各类健康记录历史数据
- WHO 胎儿生长标准参考数据

**业务规则:**
- 体重增重评估: 按孕周线性插值计算当前推荐区间
- 血压取收缩压和舒张压中较严重的等级
- 血糖根据 meal_type 取对应 GDM 阈值
- 胎动建议孕28周开始，1小时>=3次为正常
- Hadlock IV: log10(EFW) = 1.304 + 0.05281*AC(cm) + 0.1938*FL(cm) - 0.004*AC*FL
- B超 OCR: 正则提取 BPD/HC/AC/FL 数值，校验范围后供用户确认

**F6.6.1 替代食物推荐**

**F6.7.1 餐次分配设置**

**数据依赖:**

- 饮食目标（热量、模式、营养素比例）
- 食物数据库（9000+食品营养数据）
- 食物替代关系表

**业务规则:**
- 饮食模式推导: has_gdm + BMI超重 → GDM严格 / has_gdm → GDM温和 / BMI肥胖 → GDM温和 / 其他 → 均衡
- 推荐热量: BMR * 活动系数 + 孕期额外需求（孕早+0/孕中+300/孕晚+450 kcal），范围1500-2500
- 替代食物筛选: 同类别 + GI降低>=30% + 不在过敏列表 + 孕期安全
- GI分级: <=55低 / 56-69中 / >=70高

商品卡片网格: 封面图、名称、所需幸孕币、原价（划线）、已兑换人数。支持分类筛选（全部/优惠券/试用装/服务/虚拟商品）。点击进入兑换确认（校验余额/库存/限购后扣减积分）。

**F8.4.1 成就中心**
Tab 筛选（全部/已解锁/进行中/未解锁）。成就卡片展示: emoji图标、名称、描述、幸孕币奖励、进度条（如"3/7天"）。已解锁的卡片高亮显示解锁日期，进行中的显示进度百分比和"去完成"引导，未解锁的灰度展示。共8个成就: 初来乍到/健康达人/知识先锋/全勤妈妈/产检全通/饮食管家/报告解读师/胎动记录达人。

**F8.5.1 邀请有礼页**
顶部展示邀请统计（已邀请/已注册/总获幸孕币）、邀请码和二维码（可复制/保存）。奖励阶梯: 分享→注册+15币 / 首次记录+20币 / 连续签到7天+30币。分享卡片列表（宝宝发育卡/水果对照卡/产检打卡成就卡，可选择分享到微信好友/朋友圈）。邀请里程碑（3人/10人/30人额外奖励）。底部邀请记录列表（好友头像、昵称、注册日期、获得幸孕币）。FAQ 折叠面板。

**F8.6.1 个人信息编辑**
可修改: 头像（上传新头像）、昵称、就诊医院。孕期信息（LMP/EDD/体重/身高）为只读展示或跳转建档页修改。

**F8.7.1 设置页**
开关列表: 产检提醒（默认开）、每日记录提醒（默认开）、饮食提醒（默认关）、胎动提醒（默认关）、提醒时间选择（默认08:00）、数据加密（默认开）、匿名分享（默认开）。底部有"退出登录"按钮（红色文字）、"关于MOM"和"清除缓存"。

**数据依赖:**
- 用户信息（昵称、头像、MOM ID）
- 孕期档案
- 幸孕币余额、流水、每日任务完成状态
- 成就列表和进度
- 邀请码、邀请记录、邀请统计
- 商城商品列表
- 用户设置

**业务规则:**
- 幸孕币获取: 注册+100 / 建档+100 / 日记体重+10 / 日记饮食+10(每日3次) / AI提问+5(每日1次) / 阅读知识+5(每日1次) / 产检记录+20 / 报告上传+20(首次) / 预约月子+50 / 连续7天+50 / 连续30天+200
- 每日获取上限200币（注册/建档/连续打卡/成就/邀请不计入上限）
- 成就解锁后异步检查，奖励对应幸孕币并推送通知
- 邀请防作弊: 同设备只能被邀请一次 / 邀请码30天有效 / 需完成建档才算有效注册
- 兑换流程: 分布式锁防超卖 → 二次校验 → 事务扣币+减库存+创订单

## 六、版本规划建议

### 6.1 V1.0 MVP（首次上线）

**核心目标:** 验证产品核心价值，覆盖孕期管理的基础闭环。

**包含功能:**
- 微信登录 + 孕期建档（完整3步引导）
- 首页: 孕周Banner、宝宝说、妈妈健康、产检提醒、健康快捷入口
- MOM AI 助手: 智能对话、情感支持（基础版）
- 产检管理: 日历+清单视图、产检详情、记录填写、微信提醒
- 健康记录: 体重管理（含趋势图）、血压监测、血糖监测

**不包含（推后到 V1.1）:**

- 胎动计数、胎儿估重
- 饮食健康助手
- 月子规划
- 成就中心、邀请有礼、兑换商城

**核心目标:** 补齐健康监测完整链路，引入 AI 报告解读。

**规划方向:**

## 七、非功能需求

### 7.1 性能要求

| 指标 | 要求 |
|------|------|
| 首屏加载时间（FCP） | <= 1.5s（WiFi），<= 3s（4G） |
| 页面切换响应 | <= 300ms |
| API 接口响应时间 P95 | <= 500ms（常规接口） |
| API 接口响应时间 P95 | <= 3s（AI 对话首 token） |
| AI 对话流式输出 | 首 token <= 2s，总完成 <= 15s |
| OCR 识别 | <= 5s |
| 图片上传 | <= 3s（5MB 以内） |
| 离线可用 | 首页基础信息缓存，无网络时展示最近数据 |

### 7.2 安全要求

| 要求 | 说明 |
|------|------|
| 传输加密 | 全站 HTTPS，API 通信 TLS 1.2+ |
| 认证机制 | JWT Token（Access 2h + Refresh 30d） |
| 健康数据加密 | 血压、血糖等敏感数据数据库层 AES-256 加密 |
| 密钥管理 | 所有 API Key 通过环境变量注入，使用 KMS 管理 |
| 接口限流 | 每用户每分钟10次（AI/OCR 接口），防刷保护 |
| 输入校验 | 所有接口参数服务端校验，SQL 参数化查询 |
| 日志脱敏 | 日志中不记录手机号、身份证等 PII |
| 图片安全 | 上传图片内容安全检测，防色情/违规内容 |
| 隐私保护 | 用户可选匿名分享，可选数据加密开关 |
| 合规 | 用户数据收集需明确告知并获得同意，遵循《个人信息保护法》 |

### 7.3 兼容性要求

| 维度 | 要求 |
|------|------|
| 微信版本 | 微信 8.0+，基础库 2.25.0+ |
| iOS | iOS 14+ |
| Android | Android 8.0+ |
| 屏幕适配 | 375px（iPhone 8）至 430px（iPhone 15 Pro Max）宽度适配 |
| 深色模式 | V1.0 暂不支持，V1.1 适配 |
| 横屏 | 不支持，锁定竖屏 |

### 7.4 无障碍要求

| 要求 | 说明 |
|------|------|
| 文字大小 | 正文不低于12px，关键数据不低于14px |
| 色彩对比度 | 文字与背景对比度 >= 4.5:1（WCAG AA） |
| 触摸目标 | 可点击元素最小 44x44px |
| 语义化 | 页面结构使用正确的 ARIA 标签 |
| 状态反馈 | 操作结果有明确的视觉和文字反馈（Toast/Badge） |
| 异常数据颜色 | 红色高亮 + 文字描述双重提示，不仅依赖颜色 |
